# ヒトイロ（HITOIRO）— エージェント定義

> サービス名: **ヒトイロ** by goodcast / キャッチコピー: **あなたの色を、一枚に。**

## ヒアリングエージェント

### 役割
クライアントからLP制作に必要な情報を漏れなく引き出す。
加納がクライアントとの打ち合わせ中、またはヒアリングシートを元に使用する。

### 起動方法
`/hearing` または「ヒアリングして」と言う

### 動作
1. 以下の項目を順番にAskUserQuestionで質問する（一度に3〜4問ずつ）
2. 回答をまとめてJSONデータファイルを生成する

### ヒアリング項目

#### 基本情報
- お名前（フルネーム・読み仮名）
- 肩書き・職業（名乗りたい肩書き）
- キャッチコピー（一言で自分を表すなら？）
- プロフィール写真（あれば共有してもらう）

#### ストーリー
- 経歴（学歴・職歴をざっくり）
- なぜ今の仕事を始めたか（きっかけ・想い）
- 仕事で大切にしていること

#### 事業内容
- 提供しているサービス・商品（最大5つ）
- それぞれの簡単な説明
- 主なお客様（ターゲット層）

#### 実績・権威性
- メディア出演歴（TV、雑誌、Web、ラジオなど）
- 登壇・講演実績
- 受賞歴・資格
- 数字で示せる実績（○○社導入、○年の経験など）

#### お客様の声
- 推薦コメント（2〜3件）
- 推薦者の名前・肩書き（掲載許可ありのもの）

#### ビジョン
- 今後やっていきたいこと
- 実現したい世界観・ミッション

#### SNS・連絡先
- 使っているSNS（X, Instagram, YouTube, note, LinkedIn等）のURL
- 連絡手段（LINE公式、メール、Calendly、電話など）
- WebサイトURL（あれば）

#### デザインの好み
- 好きな色・ブランドカラー
- デザインの雰囲気（シンプル / ダーク高級 / カラフル / ナチュラル等）
- 参考にしたいサイトやデザイン（あれば）

### 出力
ヒアリング完了後、`src/data/clients/[slug].json` にデータを保存する。

## ライティングエージェント

### 役割
ヒアリングデータを元に、LPに掲載する魅力的なコピーを作成する。

### 起動方法
`/writing [クライアント名]` または「〇〇さんのLP文章を書いて」

### 動作
1. `src/data/clients/[slug].json` を読み込む
2. 各セクションのコピーを作成する
3. クライアントのトーン・世界観に合わせた文体にする

### ライティングルール
- **ヒーローセクション**: キャッチコピーは15文字以内。一目で「何者か」がわかる
- **ストーリー**: 3人称ではなく1人称。語りかけるような文体。300文字以内
- **サービス紹介**: 専門用語は避ける。ベネフィット（お客様が得られるもの）を中心に
- **実績**: 数字を効果的に使う。「〜社以上」「〜年」など
- **ビジョン**: 共感を生む表現。「一緒に」「ともに」を使う
- **CTA**: 行動を促す。「まずは気軽にご相談ください」など

### 出力
完成したコピーで `src/data/clients/[slug].json` を更新する。

## LP制作エージェント

### 役割
クライアントデータを元に、実際のLPページ（Next.jsコンポーネント）を生成する。

### 起動方法
`/create-lp [クライアント名]` または「〇〇さんのLPを作って」

### 動作
1. `src/data/clients/[slug].json` を読み込む
2. テーマ設定に基づいてスタイリングを決定
3. 各セクションコンポーネントを組み合わせてページを生成
4. `src/app/[slug]/page.tsx` にページを作成

### 制作ルール
- モバイルファースト（375px基準）
- レスポンシブ対応（sm / md / lg）
- ページ読み込み速度を重視（画像最適化、遅延読み込み）
- OGP（SNSシェア用画像・テキスト）を必ず設定
- Google Fonts から適切なフォントを選定
- セクション間のスクロールアニメーション（Framer Motion）

### 出力
- `src/app/[slug]/page.tsx` — メインページ
- `src/styles/themes/[slug].ts` — テーマ設定
- 必要に応じてカスタムコンポーネント

## レビューエージェント

### 役割
制作したLPの品質チェックを行う。

### 起動方法
`/review [クライアント名]` または「〇〇さんのLPをレビューして」

### チェック項目
- [ ] 全8セクションが揃っているか
- [ ] モバイル表示で崩れていないか
- [ ] リンク切れがないか
- [ ] 画像が正しく表示されるか
- [ ] OGP設定が正しいか
- [ ] ページ読み込み速度は十分か
- [ ] 誤字脱字がないか
- [ ] CTAボタンが目立つ位置にあるか
- [ ] クライアントのブランドカラーが正しく反映されているか

### 出力
レビュー結果をチェックリスト形式で報告。問題があれば修正案も提示する。

## サービスLP制作エージェント

### 役割
このサービス自体を紹介する営業用ランディングページを制作・更新する。

### 起動方法
`/service-lp` または「サービス紹介ページを作って」

### 内容
- サービスの魅力・特徴
- 料金プラン
- 制作事例（実際のクライアントLP）
- よくある質問
- お問い合わせフォーム
