# ヒアリングシート → ClientData（JSON）対応表

> `/hearing`・`/writing` がこの表に従って `src/data/clients/{slug}.json` を作る。
> JSON化の品質を担当者・実行ごとにブレさせないための基準。
> スキーマ本体は `src/lib/types.ts` の `clientDataSchema`、記入済み正解例は `src/data/clients/sample.json`。

## 基本情報

| ヒアリングシート項目 | JSONフィールド | 備考 |
|---|---|---|
| 氏名 | `name` | 表示名 |
| ふりがな | `nameKana` | Hero下部にローマ字的に表示 |
| キャッチコピー | `title` | 15文字以内。「、」で改行される |
| 肩書き・職業 | `subtitle` | Hero/メタに使用 |
| 検索されたいキーワード | `meta.keywords` | SEO用、3〜6語 |

## デザインの好み（H項目）→ theme

| ヒアリングの選択 | theme への変換 |
|---|---|
| ダーク・高級感 | `style:"dark"`／primary=濃色（#0a0a0a系）／accent=金or寒色 |
| クリーン・信頼感 | `style:"cool"`／primary=紺(#0a2540系)／accent=青(#0ea5e9) |
| ナチュラル・温かい | `style:"warm"`／primary=茶/墨／accent=橙・ベージュ／background=生成り |
| ミニマル・モノトーン | `style:"mono"`／primary=#1a1a1a／accent=控えめなグレー |
| 明るい・親しみ | `style:"light"`／background=白系／accent=ビビッド |

- 必ず `primary`（濃色背景）・`accent`（差し色）・`background`（全体背景）を決める。
- 淡色 primary を選んだ場合は `onPrimary`（濃色背景上の文字色）を明示（既定 #fafaf7）。
- 「避けたいテイスト」は配色から除外する。

## 各セクション

| ヒアリング項目 | JSONフィールド |
|---|---|
| 自己紹介・経歴ストーリー | `sections.story.heading` / `sections.story.body`（300字以内・1人称） |
| 年表・キャリア | `sections.story.career[]`（"20歳：◯◯" 形式だと数字が強調表示される） |
| 事業・サービス | `sections.services[]`（title / description） |
| 実績・メディア出演 | `sections.achievements[]`（label / detail? / date?） |
| お客様の声・推薦 | `sections.testimonials[]`（quote / author / authorTitle?）※掲載許諾書を取得 |
| ビジョン・展望 | `sections.vision.heading` / `sections.vision.body` |
| 現場写真・活動写真 | `sections.gallery[]`（src / alt / caption）※alt必須 |
| SNS各URL | `sections.sns[]`（platform / url / label?） |
| 連絡先・予約導線 | `sections.contact[]`（type / url / label）※主導線に `primary:true` |

## 画像の配置

| 用途 | JSON | 配置先 |
|---|---|---|
| メイン写真 | `sections.hero.photo` | `public/clients/{slug}/profile.jpg` 等 |
| ストーリー挿入画像 | `sections.story.breakoutImage` | `public/clients/{slug}/` |
| ギャラリー | `sections.gallery[].src` | `public/clients/{slug}/` |
| ビジョン背景 | `sections.vision.backgroundImage` | `public/clients/{slug}/` |
| OGP（SNS共有画像） | `meta.ogImage` | `public/clients/{slug}/og.png`（1200×630） |

## 必須/任意の扱い
- 中身がまだ無いセクションは**空配列 `[]`**（コンポーネントが自動で非表示）。
- 「セクション自体が不要」と「データが未回収」は案件管理シートのチェックリストで区別管理する。
