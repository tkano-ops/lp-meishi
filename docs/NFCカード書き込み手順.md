# NFCカード 書き込み・発送 手順

> 納品フェーズ【7】の物理工程。属人化しがちなノウハウを標準化する。
> カードの仕入は `docs/NFCカード仕入先リサーチ.md` を参照。

## URL 命名規則
- 形式：`{slug}.hitoiro.io`（例: `taro.hitoiro.io`）またはパス形式 `hitoiro.io/{slug}`
- slug は `src/data/clients/{slug}.json` の `slug` と一致させる。
- 公開URLが確定したら、案件管理シートに記録する。

## 書き込み手順（NTAG215）
1. **アプリ**：「NFC Tools」（iOS / Android、無料）を使用。
2. NFC Tools →「書き込み」→「レコードを追加」→「URL/URI」を選択。
3. 公開URL（`https://{slug}.hitoiro.io`）を入力。
4. 「書き込む」→ カードを端末の背面（NFCアンテナ位置）にかざす。
5. **書き込み後にロックはしない**（保守で書き換える可能性があるため）。
   - ※ 偽造・上書き防止が必要なエグゼクティブ案件のみ、合意の上でロックを検討。
6. 別の端末でかざして、正しいLPが開くことを必ず実機確認する（iPhone / Android 双方推奨）。

## 検品チェック
- [ ] かざすとLPが開く（iPhone）
- [ ] かざすとLPが開く（Android）
- [ ] QRコードからも同じURLが開く
- [ ] カード券面の氏名・ロゴ・印刷品質に問題がない

## 発送
1. 緩衝材入り封筒またはカードケースに封入。
2. 追跡可能な方法で発送（ヤマト宅急便コンパクト等）。
3. **追跡番号を `clients/{slug}/案件管理.md` に記録**。
4. 発送完了をクライアントに連絡（到着目安を添える）。

## 予備・再発行
- 書き込み済み予備カードを1枚保管しておくと、紛失時の再発行が速い。
- 再発行は月額保守の対象外（実費＋手数料）として見積る。
