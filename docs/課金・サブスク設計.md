# 課金・サブスク設計（Stripe Billing）

> 月額保守（必須）を支える課金インフラの設計メモ。
> ⚠️ 実装には Stripe アカウントと API キーが必要。**最初の数件は手動請求（銀行振込）でも回せる**ため、
> 継続課金が現実的になる直前（モニター実績が出た頃）に着手するのが妥当（提案 rank7）。

## 全体フロー
```
契約成立
  → 制作費：請求書(銀行振込) or Stripe 単発決済（着手半金／納品残金）
  → 月額保守：Stripe Billing でサブスク登録
       ↓ 毎月自動課金
     支払い成功 → LP公開継続
     支払い失敗 → リトライ → 一定期間後に LP を停止（公開停止フラグ）
  → 解約：当月分で停止、日割り返金なし
```

## Stripe 構成
- **Products**：制作費（one-time）／月額保守（recurring）
- **Prices**（`src/data/pricing.ts` と一致させる）
  - 制作費：standard ¥100,000 / premium ¥200,000 / executive ¥350,000（税抜・単発）
  - 月額：basic ¥980 / standard ¥2,980 / active ¥5,980（税抜・月次）
- **Customer**：クライアントごとに作成。`slug` を metadata に保持。
- **Webhook**：`invoice.paid` / `invoice.payment_failed` / `customer.subscription.deleted` を受ける。

## 必要な環境変数（Vercel）
```
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...
NEXT_PUBLIC_SITE_URL=https://hitoiro.co.jp
```
※ `vercel env add` で設定する。

## LP公開停止の仕組み（未払い対応）
- クライアントデータに公開フラグを持たせる案：`src/data/clients/{slug}.json` に
  `"published": false` を追加し、`[slug]/page.tsx` で false なら `notFound()`。
  （スキーマ `clientDataSchema` に `published: z.boolean().optional()` を足す）
- Webhook `invoice.payment_failed` が一定回数続いたら published=false に更新する運用。
  - JSON はリポジトリ管理のため、自動化するなら将来的に DB/KV へ移行を検討。
