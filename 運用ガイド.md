# ヒトイロ（HITOIRO）— Claude Code 運用ガイド

> サービス名: **ヒトイロ** by goodcast
> キャッチコピー: **あなたの色を、一枚に。**

## はじめに

このガイドでは、Claude Codeを使ってヒトイロ（LP名刺）を効率的に制作する方法を説明します。

---

## 1. 基本的な使い方

### 初回セットアップ（1回だけ）

```
1. VS Codeでこのフォルダを開く
2. ターミナルを開く（Ctrl + `）
3. claude と入力してEnter
4. Claude Codeが CLAUDE.md と AGENTS.md を自動で読み込む
5. 「プロジェクトを初期化して」と言えば、Next.jsアプリが作られる
```

### 日常の作業フロー

```
VS Code起動 → ターミナルで claude → 作業開始
```

Claude Codeは毎回 CLAUDE.md を読むので、プロジェクトのルールを覚えています。

---

## 2. クライアントのLP制作フロー（4ステップ）

### ステップ1: ヒアリング
```
あなた：「新しいクライアントのヒアリングをして」
```
→ Claude Codeが質問を順番に出してくれる
→ 回答していくと、クライアントデータ（JSON）が自動生成される

**使うタイミング**: クライアントとの打ち合わせ中、または打ち合わせ後にメモを元に

### ステップ2: ライティング
```
あなた：「山田さんのLP文章を書いて」
```
→ ヒアリングデータを元に、各セクションのコピーを自動生成
→ 気に入らなければ「もっとカジュアルに」「もっと権威性を出して」など指示

### ステップ3: LP制作
```
あなた：「山田さんのLPを作って」
```
→ 実際のWebページ（Next.js）が生成される
→ デザイン・色・レイアウトもクライアントに合わせて自動調整

### ステップ4: レビュー
```
あなた：「山田さんのLPをレビューして」
```
→ 品質チェック（全セクション揃ってるか、スマホ表示OK か、リンク切れないか等）

## 4. ファイル構成の説明

```
lp-meishi/
├── CLAUDE.md              ← Claude Codeへの指示書（自動で読まれる）
├── AGENTS.md              ← 役割の定義（自動で読まれる）
├── 事業計画書.md            ← 事業計画
├── 運用ガイド.md            ← このファイル
│
├── src/
│   ├── app/
│   │   ├── page.tsx        ← サービス紹介トップページ
│   │   └── [slug]/
│   │       └── page.tsx    ← 各クライアントのLPページ
│   │
│   ├── components/
│   │   └── sections/       ← LP用の部品（ヒーロー、ストーリー等）
│   │
│   ├── data/
│   │   └── clients/        ← クライアントデータ（JSON）
│   │       ├── taro-yamada.json
│   │       └── hanako-suzuki.json
│   │
│   └── styles/
│       └── themes/         ← クライアントごとの色・フォント設定
│
└── public/
    └── clients/            ← クライアントの写真素材
        ├── taro-yamada/
        └── hanako-suzuki/
```

### 重要なファイルの役割

| ファイル | 役割 | 編集するのは |
|---|---|---|
| CLAUDE.md | プロジェクトのルール | たまに更新（ルール変更時） |
| AGENTS.md | Claude Codeの役割定義 | たまに更新（フロー改善時） |
| src/data/clients/*.json | クライアント情報 | 毎回（ヒアリング後に自動生成） |
| src/app/[slug]/page.tsx | LP本体 | 毎回（自動生成 + カスタマイズ） |

---

## 5. 運用のコツ

### やっていいこと
- 「〇〇して」と日本語で普通に話しかければOK
- 「もっとこうして」「違う、こうじゃなくて」と何度でもやり直し指示
- デザインの参考URLを貼って「こんな感じにして」もOK
- CLAUDE.md や AGENTS.md の内容を改善していく

### 注意すること
- クライアントの写真は手動で `/public/clients/[slug]/` に配置する
- NFCカードの設定（URLの書き込み）は別途ツールが必要
- Vercelへのデプロイは初回だけ設定が必要

---

## 6. トラブルシューティング

| 問題 | 対処法 |
|---|---|
| Claude Codeが指示を忘れている | 新しいターミナルで `claude` を再起動 |
| デザインが気に入らない | 「もっと〇〇な感じに変えて」と具体的に指示 |
| エラーが出る | エラーメッセージをそのまま貼れば直してくれる |
| LPが表示されない | 「アプリを起動して確認して」と言う |
